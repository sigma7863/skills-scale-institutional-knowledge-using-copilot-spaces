# Role Onboarding Checklist

## Purpose
This checklist helps project managers and team leads onboard a new role or persona into an active project. Use it to ensure new team members have clarity on responsibilities, access, and communication channels.

---

## Onboarding Steps

### Project Context
- [ ] Add role contact to project one-pager or charter
- [ ] Share project background, goals, and current status
- [ ] Provide links to key project documentation (roadmap, backlog, specs)

### Role Definition
- [ ] Document role responsibilities and scope
- [ ] Add role to stakeholder list or RACI matrix
- [ ] Clarify reporting lines and escalation paths

### Coordination & Communication
- [ ] Schedule handoff meeting with Project Manager (PM)
- [ ] Schedule handoff meeting with Product Manager (PdM)
- [ ] Introduce to key team members (Developers, QA, Tech Lead, etc.)
- [ ] Add to relevant communication channels (Slack, email lists, meetings)

### Tools & Access
- [ ] Confirm access to required tools (repo, project boards, CI/CD, monitoring)
- [ ] Provide credentials or access requests for environments (dev, staging, prod)
- [ ] Share links to internal docs, wikis, or knowledge bases

### Deliverables & Expectations
- [ ] Define key deliverables and timelines
- [ ] Confirm expectations for Service Level Agreements (SLAs) or response times
- [ ] Clarify meeting attendance and reporting cadence
- [ ] Align on definition of done and quality standards

### Project Board & Tracking
- [ ] Update project board with new role as owner for relevant tasks
- [ ] Add role to sprint planning and review meetings
- [ ] Confirm access to tracking tools (Jira, GitHub Projects, etc.)

### Documentation
- [ ] Share relevant process docs (see `docs/octoacme-roles-and-personas.md`)
- [ ] Add references to role-specific templates or checklists
- [ ] Ensure access to `.copilot/` resources if using Copilot Spaces

---

## Follow-up
- [ ] Schedule 1:1 check-in after first week
- [ ] Gather feedback on onboarding experience
- [ ] Update this checklist based on lessons learned
